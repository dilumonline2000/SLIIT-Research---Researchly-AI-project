## Model details

### `trained_topic_classifier/`
- **Architecture**: TF-IDF (30,000 features, 1+2-grams) → One-vs-Rest LogReg
- **Training data**: 1,382 SLIIT papers, 80 multi-label categories
- **Output**: probability per label
- **Metrics** (15% held-out validation):
  - micro-F1 ≈ 0.15  *(strict multi-label exact-match metric)*
  - macro-F1 ≈ 0.05
  - **top-1 hit rate ≈ 0.48**  *(at least one true label in top-1 prediction)*
  - **top-3 hit rate ≈ 0.69**  *(at least one true label in top-3)*
- **Loaded by**: `app/services/topic_classifier.py`
- **Regenerate**: `python ../training/train_topic_classifier.py`

The "top-3 hit rate" metric is the operationally important one — when the
service surfaces 3 categories, ~69% of papers see at least one correct
category in the list. That's the user-visible behaviour.

### `trained_plagiarism_analyzer/`
- **What it is**: SBERT retrieval index over per-topic plagiarism statistics
- **Files**:
  - `trend_index.pkl` — `{embeddings: float32[17, 384], topics, topic_meta, version}`
  - `metadata.json` — index size, n_topics, n_records, base model
- **Reuses**: `services/module1-integrity/models/sbert_plagiarism/` as the encoder
- **Loaded by**: `app/services/plagiarism_analyzer.py`
- **Regenerate**: `python ../training/train_plagiarism_analyzer.py`

## Why these architectures?

**Topic classifier — why not SciBERT?**
- TF-IDF + LogReg trains in 4 s on CPU. SciBERT would need GPU + hours.
- 1,382 training papers is too small to fine-tune a 110M-param transformer
  effectively without overfitting.
- 80 fine-grained labels × 1,382 examples = sparse — TF-IDF surfaces strong
  lexical signals that LogReg can use without needing transfer-learning.

**Plagiarism analyzer — why retrieval, not a dedicated classifier?**
- The trend statistics are *precomputed* during training (`prepare_plagiarism_corpus.py`),
  so inference is just topic lookup. No per-query model invocation.
- The pair-comparison endpoint reuses SBERT directly — no need to re-train
  for plagiarism specifically when the SLIIT-fine-tuned encoder already
  captures paper similarity.

## Runtime expectations

| Op | Latency (CPU) | Memory |
|---|---|---|
| First load (classifier bundle) | ~600 ms | ~150 MB |
| First load (plagiarism analyzer) | ~3 s (loads SBERT) | ~250 MB |
| `categorize` query | ~50 ms | minimal |
| `summarize` (medium) on 50-sentence paper | ~80 ms | minimal |
| `plagiarism-trends/search` query | ~30 ms | minimal |
| `plagiarism-trends/compare` (2 abstracts) | ~150 ms | minimal |
| `plagiarism-trends/compare` (2 full papers) | ~1.5 s | ~50 MB |
